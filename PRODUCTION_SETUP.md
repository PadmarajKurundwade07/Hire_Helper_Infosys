# PRODUCTION DEPLOYMENT SETUP GUIDE

This is the main setup guide for deploying HireHelper to production with:
- **Frontend:** Vercel (https://hire-helper-infosys.vercel.app)
- **Backend:** Render (https://hire-helper-infosys.onrender.com)

## Quick Start

This project uses separate deployments:

1. **Backend API** runs on Render (Node.js + Express + PostgreSQL)
2. **Frontend** runs on Vercel (Angular)
3. They communicate via HTTPS REST API


## Environment Setup Summary

### Render Backend Environment Variables

Essential variables to set in Render dashboard:

```
DATABASE_URL=postgresql://user:password@host:5432/hire_helper
NODE_ENV=production
JWT_SECRET=(auto-generated by Render)
PORT=5000
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your_email@gmail.com
SMTP_PASS=your_app_password
```

### Vercel Frontend Configuration

The frontend is configured to:
1. Auto-detect and build Angular project
2. Use `/dist/frontend-app/browser` as output

## Troubleshooting

### CORS Errors

**Problem:** Frontend can't communicate with backend

**Solutions:**
1. Check backend CORS headers allow frontend domain
2. Verify backend API URL in `environment.ts`
3. Check browser console for exact error
4. Ensure backend is using HTTPS

### 500 Errors from API

**Problem:** API returns server errors

**Solutions:**
1. Check Render logs: Dashboard → **Logs** tab
2. Test `/api/health` endpoint to check database connection
3. Verify all environment variables are set in Render
4. Check database is created and accessible

### Database Connection Failed

**Problem:** Backend shows database not connected

**Solutions:**
1. Verify DATABASE_URL is set in Render
2. Check database service is running on Render
3. If using external database, verify network access
4. Check SSL is enabled for remote connections

### Deployment Fails

**Problem:** Build fails on Vercel or Render

**Solutions:**
1. Check deployment logs in dashboard
2. Run `npm install` locally to check for errors
3. Verify Node/npm versions match
4. Clear build cache and redeploy

## Deployment Workflow

1. Make changes locally and test
2. Commit to GitHub:
   ```bash
   git add .
   git commit -m "Fix: Frontend-backend integration"
   git push origin main
   ```
3. Vercel/Render automatically deploy on push
4. Monitor deployment in respective dashboards
5. Test live URLs

## Important Notes

- Frontend is a **static build** - changes require rebuild
- Backend has **automatic deployment** on code push
- Database is **persistent** - data survives redeployments
- CORS **must** be configured for frontend domain
- Use **HTTPS only** - never use HTTP in production
