================================================================================
6 SIMPLE STEPS TO DEPLOY
================================================================================

Step 1: Create Render PostgreSQL Database (5 min)
  → Go to https://render.com/dashboard
  → Create PostgreSQL service
  → Copy DATABASE_URL

Step 2: Create Render Web Service (5 min)
  → Connect GitHub repository
  → Select main branch
  → Configure Node.js environment

Step 3: Set Environment Variables (5 min)
  → Add NODE_ENV, PORT, DATABASE_URL
  → Add SMTP credentials
  → Add JWT_SECRET

Step 4: Deploy Backend (5-10 min)
  → Wait for build to complete
  → Check status = "Live"

Step 5: Verify Health Check (2 min)
  → Visit /api/health endpoint
  → Confirm database: { connected: true }

Step 6: Test Production (5 min)
  → Register at frontend URL
  → Verify OTP email
  → Login and access dashboard

TOTAL TIME: ~30-40 minutes

QUICK REFERENCE COMMANDS
================================================================================

Test Backend Health:
  curl https://your-backend-url.onrender.com/api/health

Test Backend Root:
  curl https://your-backend-url.onrender.com/

Test Registration (local):
  curl -X POST http://localhost:5000/api/auth/register \
    -H "Content-Type: application/json" \
    -d '{"first_name":"John","last_name":"Doe","email_id":"test@gmail.com","password":"Pass123","phone_number":"1234567890"}'

Local Development:
  Backend: cd backend && npm run dev
  Frontend: cd frontend && ng serve
  Browser: http://localhost:4200

View Backend Logs (Render):
  https://dashboard.render.com → Your Service → Logs tab

View Frontend Logs (Vercel):
  https://vercel.com/dashboard → Your Project → Deployments
